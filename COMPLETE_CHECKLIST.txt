If any test fails, use this troubleshooting guide:

Issue: Port already in use (e.g., "Address already in use")
  Step 1: Find the process using the port
    netstat -ano | findstr :8545 (or :8000, :3000)
  Step 2: Get the PID from the results
  Step 3: Kill the process
    taskkill /PID <PID> /F
  Step 4: Try starting again

Issue: Ganache not starting
  Step 1: Verify Ganache is installed
    npm list -g ganache-cli
  Step 2: Install if missing
    npm install -g ganache-cli
  Step 3: Try starting again
    ganache-cli -p 8545 -d

Issue: Smart contract deployment fails
  Step 1: Verify Ganache is running (check first terminal)
  Step 2: Verify truffle-config.js has correct port (should be 8545)
  Step 3: Try again
    truffle migrate --reset
  Step 4: If still fails, restart Ganache and try again

Issue: Django can't connect to blockchain
  Check Django terminal for: "eth_abi.exceptions.InsufficientDataBytes"
  Step 1: Verify Ganache is running
  Step 2: Verify contract is deployed
  Step 3: Check Django logs for exact error
  Step 4: Restart Django
  Step 5: Check blockchain connection re-established

Issue: Certificate shows as Invalid âŒ
  Check: Is blockchain running?
  Check: Is contract deployed?
  Step 1: Give it 2-3 seconds after issuing
  Step 2: Refresh browser (Ctrl+R)
  Step 3: Check Django logs for verification messages
  Step 4: Try issuing a NEW certificate
  Step 5: Verify immediately after issuance

Issue: QR upload returns 500 error
  Check: File size (must be < 5MB)
  Check: File format (must be image: .png, .jpg, .gif)
  Step 1: Check Django logs for detailed error
  Step 2: Try different QR image
  Step 3: Verify Pillow is installed
    pip list | findstr Pillow
  Step 4: Verify pyzbar is installed
    pip list | findstr pyzbar
  Step 5: If missing, install and restart Django
